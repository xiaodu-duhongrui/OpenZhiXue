# 学生端移动应用 - 产品需求文档

## 概述
- **摘要**: 基于已完成的 Flutter 移动端基础设施（Task 16），开发面向学生的移动应用，提供课程浏览、作业提交、成绩查询、消息通知、个人中心等核心功能。
- **目的**: 为学生提供便捷的移动端学习管理工具，支持随时随地查看课程、完成作业、查询成绩、接收通知，提升学习效率。
- **目标用户**: 在校学生（初中、高中学生群体）

## 目标
- 提供完整的课程浏览和学习体验
- 实现作业查看、提交和状态跟踪功能
- 支持成绩查询和趋势分析
- 提供实时消息通知功能
- 提供完善的个人中心管理功能
- 确保应用在主流移动设备上的兼容性和性能

## 非目标（范围外）
- 教师端和家长端功能（由 Task 18、Task 19 负责）
- 在线考试功能（本次迭代不包含）
- 视频播放功能（本次迭代仅展示课程列表和基本信息）
- 社交功能（同学互动、评论等）

## 背景与上下文
- Task 16 已完成 Flutter 项目初始化，包括：
  - 多平台支持（iOS/Android/鸿蒙）
  - 网络请求层（Dio + 拦截器）
  - 状态管理（Riverpod）
  - 路由管理（GoRouter）
  - 本地存储（SharedPreferences + SecureStorage）
  - UI 组件库和主题系统
- 后端服务已完成开发：
  - 用户认证服务（Go）
  - 学生管理服务（Python）
  - 成绩管理服务（Python）
  - 作业管理服务（Rust）
  - 考试管理服务（Go）
  - 在线学习服务（Python）
  - 家校沟通服务（Java）

## 功能需求

### FR-1: 首页仪表盘
- **FR-1.1**: 显示欢迎卡片，包含用户姓名和今日待办摘要
- **FR-1.2**: 显示快捷功能入口（作业、成绩、课程、消息）
- **FR-1.3**: 显示最近作业列表（最多5条），支持状态标签
- **FR-1.4**: 支持下拉刷新功能

### FR-2: 作业功能模块
- **FR-2.1**: 作业列表页面，支持按课程、状态筛选
- **FR-2.2**: 作业详情页面，显示作业要求、截止时间、附件
- **FR-2.3**: 作业提交功能，支持文本内容和附件上传
- **FR-2.4**: 作业状态显示（待完成、已提交、已批改、已逾期）
- **FR-2.5**: 批改结果查看，包含分数和教师反馈

### FR-3: 成绩查询模块
- **FR-3.1**: 成绩列表页面，支持按科目、考试筛选
- **FR-3.2**: 成绩详情页面，显示分数、班级排名、年级排名
- **FR-3.3**: 成绩趋势图表，展示近期考试成绩变化
- **FR-3.4**: 成绩统计摘要（平均分、最高分、最低分）

### FR-4: 课程学习模块
- **FR-4.1**: 课程列表页面，显示本学期所有课程
- **FR-4.2**: 课程详情页面，显示课程信息、教师信息
- **FR-4.3**: 课程表视图，按周展示课程安排
- **FR-4.4**: 课时列表，显示课程章节和课时

### FR-5: 消息通知模块
- **FR-5.1**: 消息列表页面，显示所有通知消息
- **FR-5.2**: 消息分类（系统通知、作业提醒、成绩通知）
- **FR-5.3**: 消息详情页面
- **FR-5.4**: 未读消息计数和已读标记
- **FR-5.5**: 推送通知支持

### FR-6: 个人中心模块
- **FR-6.1**: 个人信息展示和编辑
- **FR-6.2**: 头像上传功能
- **FR-6.3**: 设置页面（主题切换、语言设置、通知设置）
- **FR-6.4**: 账号安全设置（修改密码）
- **FR-6.5**: 关于页面（版本信息、用户协议、隐私政策）
- **FR-6.6**: 退出登录功能

## 非功能需求

### NFR-1: 性能要求
- 应用启动时间不超过 3 秒
- 页面切换流畅，帧率不低于 60fps
- 列表滚动无明显卡顿
- API 请求响应时间不超过 2 秒（正常网络环境）

### NFR-2: 兼容性要求
- 支持 iOS 12.0 及以上版本
- 支持 Android 6.0 (API 23) 及以上版本
- 支持鸿蒙 HarmonyOS 3.0 及以上版本
- 适配主流屏幕尺寸（4.7寸 - 7寸）

### NFR-3: 安全性要求
- 用户敏感数据（Token）使用 SecureStorage 加密存储
- 网络请求使用 HTTPS 加密传输
- 敏感操作需要用户确认

### NFR-4: 离线支持
- 已查看的数据支持本地缓存
- 离线时显示缓存数据并提示网络状态
- 网络恢复后自动同步数据

### NFR-5: 可访问性
- 支持系统字体缩放
- 支持深色模式
- 颜色对比度符合 WCAG 2.0 AA 标准

## 约束条件
- **技术约束**: 基于 Flutter 3.2+ 开发，使用 Riverpod 状态管理
- **依赖约束**: 依赖 Task 16 已完成的基础设施
- **API 约束**: 需对接现有后端微服务 API

## 假设
- 后端 API 已完成并可供调用
- 用户已通过认证服务完成登录
- 移动设备已连接网络（部分功能支持离线）

## 验收标准

### AC-1: 首页仪表盘
- **Given**: 学生用户已登录应用
- **When**: 进入首页
- **Then**: 显示欢迎卡片、快捷功能入口和最近作业列表
- **验证方式**: `human-judgment`

### AC-2: 作业列表
- **Given**: 学生用户进入作业页面
- **When**: 查看作业列表
- **Then**: 显示所有作业，支持按状态和课程筛选
- **验证方式**: `programmatic`

### AC-3: 作业提交
- **Given**: 学生查看未完成的作业详情
- **When**: 填写内容并点击提交
- **Then**: 作业状态更新为已提交，显示提交成功提示
- **验证方式**: `programmatic`

### AC-4: 成绩查询
- **Given**: 学生用户进入成绩页面
- **When**: 查看成绩列表
- **Then**: 显示所有考试成绩，支持筛选和查看详情
- **验证方式**: `programmatic`

### AC-5: 成绩趋势图
- **Given**: 学生查看成绩详情
- **When**: 查看趋势分析
- **Then**: 显示近期考试成绩变化趋势图
- **验证方式**: `human-judgment`

### AC-6: 课程列表
- **Given**: 学生用户进入课程页面
- **When**: 查看课程列表
- **Then**: 显示本学期所有课程，支持查看详情
- **验证方式**: `programmatic`

### AC-7: 课程表
- **Given**: 学生用户查看课程表
- **When**: 切换周次
- **Then**: 显示对应周次的课程安排
- **验证方式**: `programmatic`

### AC-8: 消息通知
- **Given**: 学生收到新消息
- **When**: 打开消息列表
- **Then**: 显示未读消息计数，消息按时间排序
- **验证方式**: `programmatic`

### AC-9: 个人中心
- **Given**: 学生进入个人中心
- **When**: 查看或编辑个人信息
- **Then**: 显示当前用户信息，支持编辑和保存
- **验证方式**: `programmatic`

### AC-10: 离线缓存
- **Given**: 用户已查看过某些数据
- **When**: 设备离线时再次查看
- **Then**: 显示缓存数据并提示离线状态
- **验证方式**: `programmatic`

### AC-11: 应用性能
- **Given**: 应用在主流设备上运行
- **When**: 进行页面切换和列表滚动
- **Then**: 启动时间 < 3秒，帧率 >= 60fps
- **验证方式**: `programmatic`

### AC-12: 兼容性测试
- **Given**: 应用在不同平台和设备上运行
- **When**: 执行功能测试
- **Then**: 在 iOS、Android、鸿蒙平台上功能正常
- **验证方式**: `programmatic`
